# Deployment Guide: Student Council Election & Portal System

This guide provides the necessary steps to deploy and configure the Student Council Portal.

## 1. Supabase Database Setup

Your application uses Supabase for data storage. Follow these steps to prepare your database:

### Create Tables and Initial Data
Open the **SQL Editor** in your Supabase dashboard and execute the following script:

```sql
-- Create Tables
CREATE TABLE settings (id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY, key TEXT UNIQUE NOT NULL, value TEXT NOT NULL);
CREATE TABLE students (id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY, student_number TEXT UNIQUE NOT NULL, name TEXT NOT NULL, year TEXT, section TEXT, has_voted BOOLEAN DEFAULT FALSE, created_at TIMESTAMPTZ DEFAULT NOW());
CREATE TABLE home_content (id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY, section_key TEXT UNIQUE NOT NULL, title TEXT NOT NULL, content TEXT NOT NULL, order_index INT DEFAULT 0);
CREATE TABLE news (id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY, title TEXT NOT NULL, content TEXT NOT NULL, date TEXT NOT NULL, image_url TEXT, created_at TIMESTAMPTZ DEFAULT NOW());
CREATE TABLE officers (id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY, name TEXT NOT NULL, position TEXT NOT NULL, year TEXT NOT NULL, image_url TEXT, is_current BOOLEAN DEFAULT TRUE, category TEXT DEFAULT 'Executive', created_at TIMESTAMPTZ DEFAULT NOW());
CREATE TABLE memories (id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY, image_url TEXT NOT NULL, caption TEXT, batch TEXT, created_at TIMESTAMPTZ DEFAULT NOW());
CREATE TABLE terms (id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY, name TEXT NOT NULL, is_active BOOLEAN DEFAULT FALSE);
CREATE TABLE partylists (id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY, name TEXT UNIQUE NOT NULL, platform_image_url TEXT);
CREATE TABLE positions (id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY, name TEXT UNIQUE NOT NULL, category TEXT DEFAULT 'Executive');
CREATE TABLE candidates (id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY, name TEXT NOT NULL, position TEXT NOT NULL, grade_level TEXT, partylist_id BIGINT REFERENCES partylists(id) ON DELETE SET NULL, category TEXT, term_id BIGINT REFERENCES terms(id) ON DELETE CASCADE, voting_restriction TEXT DEFAULT 'everyone', image_url TEXT, created_at TIMESTAMPTZ DEFAULT NOW());
CREATE TABLE votes (id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY, student_id BIGINT REFERENCES students(id) ON DELETE CASCADE, candidate_id BIGINT REFERENCES candidates(id) ON DELETE CASCADE, position TEXT NOT NULL, created_at TIMESTAMPTZ DEFAULT NOW());
CREATE TABLE sections (id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY, year TEXT NOT NULL, name TEXT NOT NULL);
CREATE TABLE inquiries (id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY, name TEXT NOT NULL, email TEXT NOT NULL, subject TEXT NOT NULL, message TEXT NOT NULL, created_at TIMESTAMPTZ DEFAULT NOW());
CREATE TABLE suggestions (id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY, category TEXT NOT NULL, content TEXT NOT NULL, is_anonymous BOOLEAN DEFAULT TRUE, student_id BIGINT REFERENCES students(id) ON DELETE SET NULL, created_at TIMESTAMPTZ DEFAULT NOW());

-- Insert Default Configuration
INSERT INTO settings (key, value) VALUES ('admin_password', 'admin123');
INSERT INTO settings (key, value) VALUES ('voting_restriction', 'everyone');
INSERT INTO settings (key, value) VALUES ('logo1', ''), ('logo2', '');

-- Disable RLS (Row Level Security) for initial setup
-- You can enable this later and add specific policies if needed.
ALTER TABLE settings DISABLE ROW LEVEL SECURITY;
ALTER TABLE students DISABLE ROW LEVEL SECURITY;
ALTER TABLE home_content DISABLE ROW LEVEL SECURITY;
ALTER TABLE news DISABLE ROW LEVEL SECURITY;
ALTER TABLE officers DISABLE ROW LEVEL SECURITY;
ALTER TABLE memories DISABLE ROW LEVEL SECURITY;
ALTER TABLE terms DISABLE ROW LEVEL SECURITY;
ALTER TABLE partylists DISABLE ROW LEVEL SECURITY;
ALTER TABLE positions DISABLE ROW LEVEL SECURITY;
ALTER TABLE candidates DISABLE ROW LEVEL SECURITY;
ALTER TABLE votes DISABLE ROW LEVEL SECURITY;
ALTER TABLE sections DISABLE ROW LEVEL SECURITY;
ALTER TABLE inquiries DISABLE ROW LEVEL SECURITY;
ALTER TABLE suggestions DISABLE ROW LEVEL SECURITY;
```

### Storage Configuration
1. Go to **Storage** in your Supabase dashboard.
2. Create a new bucket named **`uploads`**.
3. Set the bucket privacy to **Public**.

## 2. Environment Variables

Configure the following environment variables in your hosting environment:

| Variable | Description |
| :--- | :--- |
| `VITE_SUPABASE_URL` | Your Supabase Project URL (e.g., `https://xyz.supabase.co`) |
| `VITE_SUPABASE_ANON_KEY` | Your Supabase `anon` public key |
| `SUPABASE_SERVICE_ROLE_KEY` | Your Supabase `service_role` secret key (Required for Admin functions) |

## 3. Deployment Options

### Option A: Netlify (Recommended for Static Hosting)
The application has been updated to be a **Pure Single Page Application (SPA)**. This means it talks directly to Supabase from the browser, making it compatible with Netlify.

1. Connect your GitHub repository to Netlify.
2. Set the **Build Command** to `npm run build`.
3. Set the **Publish Directory** to `dist`.
4. Add your **Environment Variables** in the Netlify UI.
5. **Important**: Since the app is an SPA, you must add a `_redirects` file to the `public` folder with the following content to handle routing:
   ```
   /*  /index.html  200
   ```

### Option B: Railway / Render (Recommended for Full-Stack)
If you want to use the Express server, use a host that supports Node.js.
1. Connect your repository.
2. The platform will automatically detect the `start` script and run the server.

---

## 4. Initial Access

- **Admin Dashboard**: Access the `/admin` route and use the default password: `admin123`.
- **Changing Password**: Once logged in, navigate to the **Settings** tab to update the admin password.
- **Student Login**: Students log in using their **Student Number**. Ensure you have uploaded or added student records via the Admin Dashboard before testing student login.

---

## 5. Common Issues

- **Invalid API Key**: Double-check that your Supabase URL and keys are correct. Ensure the keys do not have any extra prefixes or spaces.
- **Database Errors**: Ensure the SQL script was executed successfully and all tables exist.
- **File Uploads Failing**: Verify that the `uploads` storage bucket exists and is set to **Public**.
